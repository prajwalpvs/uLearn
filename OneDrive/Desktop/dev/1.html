Two Sum
Easy
Hash Map · Arrays
Given an array of integers and a target, return indices of two numbers that add up to the target.
► show solution
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []
Time: O(n)
Space: O(n)
Valid Parentheses
Easy
Stack · Strings
Determine if a string of brackets ()[]{} is valid (properly opened and closed).
► show solution
def is_valid(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in '([{':
            stack.append(ch)
        else:
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack
Time: O(n)
Space: O(n)
Reverse Linked List
Easy
Linked List · Pointers
Reverse a singly linked list iteratively in-place.
► show solution
def reverse(head):
    prev = None
    curr = head
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    return prev
Time: O(n)
Space: O(1)
Best Time to Buy/Sell Stock
Easy
Greedy · Arrays
Given prices array, find max profit from one buy + one sell.
► show solution
def max_profit(prices):
    min_price = float('inf')
    profit = 0
    for p in prices:
        min_price = min(min_price, p)
        profit = max(profit, p - min_price)
    return profit
Time: O(n)
Space: O(1)
Longest Substring No Repeat
Medium
Sliding Window · Hash Set
Find the length of the longest substring with no repeating characters.
► show solution
def length_of_longest(s):
    seen = {}
    left = max_len = 0
    for right, ch in enumerate(s):
        if ch in seen and seen[ch] >= left:
            left = seen[ch] + 1
        seen[ch] = right
        max_len = max(max_len, right - left + 1)
    return max_len
Time: O(n)
Space: O(min(n, charset))
LRU Cache
Medium
Hash Map · Doubly Linked List
Implement Least Recently Used cache with O(1) get and put operations.
► show solution
from collections import OrderedDict

class LRUCache:
    def __init__(self, cap):
        self.cap = cap
        self.cache = OrderedDict()

    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]

    def put(self, key, val):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = val
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)
Time: O(1) get/put
Space: O(capacity)
Number of Islands
Medium
DFS / BFS · Grid
Count islands in a 2D grid of '1's (land) and '0's (water).
► show solution
def num_islands(grid):
    if not grid: return 0
    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r, c):
        if r < 0 or c < 0 or r >= rows \
           or c >= cols or grid[r][c] != '1':
            return
        grid[r][c] = '0'
        dfs(r+1, c); dfs(r-1, c)
        dfs(r, c+1); dfs(r, c-1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                dfs(r, c)
    return count
Time: O(m·n)
Space: O(m·n) recursion
Merge K Sorted Lists
Hard
Heap · Linked List
Merge k sorted linked lists into one sorted list.
► show solution
import heapq

def merge_k_lists(lists):
    heap = []
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    dummy = curr = ListNode()
    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
Time: O(n log k)
Space: O(k)
Word Break
Medium
Dynamic Programming
Given a string and dictionary, can the string be segmented into dictionary words?
► show solution
def word_break(s, words):
    word_set = set(words)
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True

    for i in range(1, n + 1):
        for j in range(i):
            if dp[j] and s[j:i] in word_set:
                dp[i] = True
                break
    return dp[n]
Time: O(n²)
Space: O(n)
Trapping Rain Water
Hard
Two Pointers · Arrays
Given heights, compute how much water can be trapped after raining.
► show solution
def trap(h):
    left, right = 0, len(h) - 1
    lmax = rmax = water = 0
    while left < right:
        if h[left] < h[right]:
            if h[left] >= lmax: lmax = h[left]
            else: water += lmax - h[left]
            left += 1
        else:
            if h[right] >= rmax: rmax = h[right]
            else: water += rmax - h[right]
            right -= 1
    return water
Time: O(n)
Space: O(1)
Find Duplicate Number
Medium
Floyd's Cycle · Math
Array of n+1 ints in [1,n] has one duplicate. Find it without modifying array, O(1) space.
► show solution
def find_duplicate(nums):
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast: break

    slow = nums[0]
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    return slow
Time: O(n)
Space: O(1)
Producer-Consumer
Medium
Concurrency · Threads
Classic synchronization — producers add to buffer, consumers remove. No race conditions.
► show solution
import threading, queue, time

buffer = queue.Queue(maxsize=5)

def producer():
    for i in range(10):
        buffer.put(i)
        print(f"Produced {i}")
        time.sleep(0.1)

def consumer():
    while True:
        item = buffer.get()
        print(f"Consumed {item}")
        buffer.task_done()

threading.Thread(target=producer).start()
threading.Thread(target=consumer, daemon=True).start()
Note: queue.Queue is thread-safe
Binary Search
Easy
Binary Search · Arrays
Given a sorted array and a target, return the index of the target or -1 if not found. O(log n) required.
► show solution
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
Time: O(log n)
Space: O(1)
Merge Intervals
Medium
Sorting · Arrays
Given a list of intervals, merge all overlapping ones and return the non-overlapping result.
► show solution
def merge(intervals):
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])
    return merged
Time: O(n log n)
Space: O(n)
Coin Change
Medium
Dynamic Programming
Given coin denominations and an amount, return the fewest coins needed to make that amount, or -1 if impossible.
► show solution
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for amt in range(coin, amount + 1):
            dp[amt] = min(dp[amt], dp[amt - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
Time: O(amount × coins)
Space: O(amount)
Kth Largest Element
Medium
QuickSelect · Heap
Find the kth largest element in an unsorted array. Beat O(n log n) sort with QuickSelect O(n) average.
► show solution
import random

def find_kth_largest(nums, k):
    pivot = random.choice(nums)
    greater = [x for x in nums if x > pivot]
    equal   = [x for x in nums if x == pivot]
    less    = [x for x in nums if x < pivot]
    if k <= len(greater):
        return find_kth_largest(greater, k)
    if k <= len(greater) + len(equal):
        return pivot
    return find_kth_largest(less, k - len(greater) - len(equal))
Time: O(n) avg, O(n²) worst
Space: O(n)

TCP vs UDP

TCP — Transmission Control Protocol

  • Connection-oriented (SYN → SYN-ACK → ACK handshake)
  • Guaranteed delivery, in-order, error-checked
  • Flow control + congestion control built in
  • Higher overhead — use when reliability matters
  • Use: HTTP, SSH, databases, email (SMTP)

UDP — User Datagram Protocol

  • Connectionless — fire and forget, no handshake
  • No delivery guarantee, no ordering
  • Very low overhead and latency
  • App handles packet loss if needed
  • Use: DNS, video streaming, gaming, VoIP, HTTP/3 (QUIC)

  • TTL controls cache duration — lower TTL means faster propagation but more resolver load
  • Record types: A (→ IPv4), AAAA (→ IPv6), CNAME (alias → canonical name), MX (mail server), TXT (SPF/DKIM)
  • DoH (DNS over HTTPS) and DoT (DNS over TLS) encrypt DNS queries to prevent eavesdropping
HTTP Evolution
  • HTTP/1.1 — persistent connections but HOL blocking; effectively 1 in-flight request per connection despite pipelining
  • HTTP/2 — binary framing; multiplexing (many streams per TCP conn); header compression (HPACK); server push
  • HTTP/3 — runs on QUIC (UDP-based); eliminates TCP HOL blocking; 0-RTT session resumption; built-in TLS 1.3; better on lossy networks
TLS 1.3 Handshake
  • ClientHello — supported ciphers, DH key share, random nonce
  • ServerHello — chosen cipher, server key share, certificate chain
  • Client verifies cert against trusted CA roots in browser/OS store
  • Session keys derived via Diffie-Hellman — secret never transmitted
  • TLS 1.3: 1 RTT total (vs 2 for TLS 1.2); 0-RTT for session resumption
  • PFS: ephemeral keys mean past traffic can't be decrypted if private key leaks later
WebSockets vs SSE vs Polling
  • WebSocket — full-duplex, persistent; upgrades from HTTP via Upgrade: websocket; use for chat, games, live collab
  • SSE (Server-Sent Events) — server-to-client only; uses plain HTTP; simpler; reconnects automatically; use for dashboards, notifications
  • Long polling — client holds request open; server replies when data ready; fallback for restricted environments
  • Scale WebSockets horizontally with sticky sessions or shared Redis pub/sub
Load Balancers: L4 vs L7
  • L4 (Transport) — routes by IP + TCP/UDP port; no content inspection; very fast; e.g. AWS NLB, HAProxy TCP
  • L7 (Application) — routes by URL path, headers, cookies; SSL termination; A/B routing; e.g. AWS ALB, nginx, Envoy
  • Algorithms: Round Robin, Least Connections, IP Hash (sticky), Weighted RR
  • Health checks: periodic probes remove unhealthy backends automatically
  • CDN (Cloudflare, CloudFront) = globally distributed L7 cache + anycast edge routing
Authentication vs Authorization

Authentication (AuthN) — Who are you?

  • Prove identity: password, biometric, hardware key, OTP
  • Result: session cookie, JWT, signed token
  • Protocols: OAuth 2.0, OIDC, SAML, LDAP
  • Factors: something you know / have / are → MFA

Authorization (AuthZ) — What can you do?

  • Checks permissions after identity confirmed
  • RBAC — roles (admin, editor, viewer)
  • ABAC — attributes (dept=eng, clearance=3)
  • Principle of Least Privilege — grant minimum needed
  • Tools: OPA (Open Policy Agent), Casbin, AWS IAM
JWT — JSON Web Tokens
Structure: header.payload.signature (all base64url encoded)
// Header — algorithm + token type
{ "alg": "RS256", "typ": "JWT" }

// Payload — claims (public + private)
{ "sub": "user_123", "role": "admin", "exp": 1735689600 }

// Signature — server verifies on every request
RSASHA256(base64url(header) + "." + base64url(payload), privateKey)
  • Stateless — server validates signature without DB lookup (ideal for microservices)
  • Prefer RS256 (asymmetric) over HS256 (shared secret) for distributed systems
  • Short expiry (15 min access token) + long-lived refresh token + rotation = revocability
  • Store in HttpOnly cookie, NOT localStorage — prevents XSS token theft
  • Payload is base64-encoded, not encrypted — never put secrets in JWT claims
OAuth 2.0 Grant Types
Authorization Code + PKCE (web / mobile)
  • User authenticates at AuthServer → redirected back with a one-time code
  • Client exchanges code + code_verifier (PKCE) for access + refresh tokens
  • PKCE prevents code interception attacks — safe for public clients (SPA, mobile)
  • Never expose client_secret in frontend JS — use PKCE instead
Client Credentials (Machine-to-Machine)
  • Service sends client_id + client_secret directly to AuthServer for token
  • No user involved — for backend-to-backend API calls
  • Rotate secrets via secrets manager; never hardcode in source code
  • Scope access narrowly — one client per service per environment
Password Hashing
Use bcrypt / Argon2id / scrypt — never raw MD5, SHA1, or SHA256
import bcrypt

# Hash on registration (cost=12 ≈ 300ms — slow is the point)
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt(rounds=12))

# Verify on login — uses constant-time comparison internally
is_valid = bcrypt.checkpw(candidate.encode(), hashed)
  • bcrypt automatically salts each hash — no separate salt management needed
  • Argon2id is the 2023 OWASP recommendation — memory-hard, resistant to GPU cracking
  • MD5 / SHA-256 are fast checksums for data integrity, not designed for passwords
  • Increase cost factor over time as hardware improves; re-hash on next login
OWASP Top 10 Essentials
SQL Injection
# Vulnerable — attacker passes id="1 OR 1=1"
query = f"SELECT * FROM users WHERE id={user_id}"

# Safe — parameterized query
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))

Use ORMs or parameterized queries everywhere. Grant DB accounts only the permissions they need. Validate and whitelist all user input at the boundary.

XSS — Cross-Site Scripting
// Vulnerable — injects attacker JS into DOM
element.innerHTML = userInput;

// Safe — treats as plain text
element.textContent = userInput;
// Rich text: sanitize with DOMPurify first

Escape all user-controlled output in templates. Set Content-Security-Policy header. Use HttpOnly cookies so JS cannot read auth tokens even if XSS occurs.

CSRF — Cross-Site Request Forgery
  • Attacker's page tricks logged-in user's browser into sending a forged request to your API
  • Defense 1: CSRF token — random, per-session, validated server-side on every mutating request
  • Defense 2: SameSite=Strict or SameSite=Lax cookie attribute blocks cross-origin sends
  • Defense 3: Verify Origin / Referer headers on state-changing endpoints
Cookie Security Flags + CORS
  • HttpOnly — JS cannot read cookie; blocks XSS token theft
  • Secure — only sent over HTTPS; never over HTTP
  • SameSite=Strict — never sent cross-site (may break OAuth redirects)
  • SameSite=Lax — sent on top-level navigation only (safe default)
  • CORS: Access-Control-Allow-Origin restricts which origins can call your API; never use * with credentials
The Test Pyramid

Layer Overview (bottom = most, top = fewest)

  • 🟢 Unit — fast, isolated, mock all deps. Run on every save.
  • 🔵 Integration — real components together (DB, queue, API). Run in CI.
  • 🔴 E2E — real browser, full stack, slow and flaky. Run before release.

Rule of Thumb

  • 70% unit / 20% integration / 10% E2E
  • More integration = more confidence, slower CI
  • Test behavior, not implementation details
  • Don't mock what you don't own — use fakes instead
  • "Test the contract, not the internals"
TDD — Red → Green → Refactor
🔴 Write failing test
→
🟢 Minimal code to pass
→
🔵 Refactor safely
→
repeat
  • Forces API design before implementation — surfaces awkward interfaces early
  • Tests become living documentation of intended behavior
  • Don't TDD everything — spikes, UI exploration, migrations rarely benefit
  • Name tests: test_[unit]_when_[scenario]_should_[expected] for clarity
Test Doubles
Mock — verifies behavior
from unittest.mock import Mock
mailer = Mock()
register_user(user, mailer)
mailer.send.assert_called_once_with("user@x.com")

Pre-programmed with call expectations. Fails the test if not invoked correctly. Overuse leads to brittle tests tightly coupled to implementation.

Stub — returns canned data
def fake_get_user(id):
    return {'id': id, 'name': 'Alice'}

monkeypatch.setattr(db, 'get_user', fake_get_user)

Returns fixed responses; does not verify call count. Simpler and less brittle than mocks. Used for state-based testing where you care about the result, not the path.

Fake — working lightweight implementation
  • Working implementation — simpler than production (e.g. in-memory DB, fake payment processor)
  • More realistic than stubs; actually maintains state across calls
  • Best for replacing infrastructure dependencies in test suites
  • Example: SQLite in tests, fake S3 via localstack or moto
Coverage — What It Misses
  • 100% line coverage does NOT mean no bugs — logic errors still slip through
  • Branch coverage matters more — every if/else/case path needs a test
  • Mutation testing (mutmut, Pitest) verifies your tests actually catch real bugs
  • Coverage theater: an assertion that always passes can hit 100% coverage
  • Aim for 80%+ meaningful coverage with real assertions, not vanity metrics

  • Private subnet: no direct internet; for DBs, internal services; use NAT gateway for outbound

IAM — Identity and Access Management
Core Concepts: Principal → Policy → Role
# AWS IAM policy — explicit allow on specific resource
{
  "Effect": "Allow",
  "Action": ["s3:GetObject", "s3:PutObject"],
  "Resource": "arn:aws:s3:::my-bucket/*",
  "Condition": { "StringEquals": { "s3:prefix": "uploads/" } }
}
  • Principal — who: IAM user, role, service account, federated identity
  • Policy — what actions are allowed/denied on which resources under what conditions
  • Role — named policy bundle; assumed by EC2 instances, Lambda functions, CI/CD pipelines
  • Use instance profiles / workload identity — never hardcode credentials in code or env files
  • Prefer temporary credentials via STS AssumeRole over long-lived access keys
  • Enable MFA on all IAM users, especially root; delete root access keys immediately after setup
  • Audit regularly with IAM Access Analyzer — automatically surfaces overly permissive policies
Serverless vs Containers

Serverless (Lambda / Cloud Functions)

  • Zero server management; auto-scales to millions of requests
  • Pay per invocation + duration (ms granularity)
  • Cold start: 100ms–2s on first invocation (use Provisioned Concurrency to eliminate)
  • Max execution: 15 min (Lambda); no persistent connections
  • Best for: event handlers, webhooks, scheduled jobs, glue code, bursty workloads

Containers (ECS / GKE / AKS)

  • Full control over runtime, dependencies, OS packages
  • Predictable latency; no cold starts; supports persistent connections
  • Pay per running instance even when idle
  • Best for: long-running APIs, WebSocket servers, ML inference, high-throughput services
  • Ops overhead: health checks, autoscaling config, rolling deploys, image scanning
